# Тестирование внутриигрового магазина и систем монетизации

## Обзор

Данный Case Study демонстрирует полный процесс QA-тестирования внутриигрового магазина и систем монетизации мобильной игры.

Основной фокус был направлен не только на проверку функциональности, но и на поиск рисков, влияющих на:

- пользовательский опыт
- стабильность монетизации
- систему наград
- продуктовые показатели
- качество релиза

---

# Цели тестирования

Основные цели:

- проверить стабильность процесса покупки
- проверить корректную выдачу наград
- проверить работу специальных предложений
- оценить пользовательский опыт
- найти нестандартные критические сценарии
- проверить корректность аналитики
- снизить риски перед релизом

---

# Подход к тестированию

Использовались:

- функциональное тестирование
- исследовательское тестирование
- риск-ориентированное тестирование
- регрессионное тестирование
- проверка аналитики

Основное внимание уделялось:

- покупкам
- наградам
- предложениям
- прогрессии игрока
- процессам монетизации

# Документация

В данный Case Study входят:

📄 **Case Study**

Анализ функциональности, стратегия тестирования, риски, найденные проблемы и результат.

📋 **Тест-план**

Область тестирования, цели, окружение, критерии начала и завершения тестирования.

✅ **Тест-кейсы**

Покрытие основных сценариев, негативных проверок и нестандартных случаев.

🐞 **Баг-репорты**

Описание найденных проблем с приоритетом, серьёзностью и влиянием на продукт.

🔍 **Отчёт исследовательского тестирования**

Анализ нестандартных сценариев и потенциальных рисков.

---

# Основные проверенные риски

- блокировка процесса покупки
- некорректная выдача наград
- конфликты интерфейса монетизации
- ошибки логики предложений
- проблемы аналитики

---

# Результат

Тестирование помогло:

- предотвратить критические проблемы монетизации
- улучшить стабильность пользовательского опыта
- снизить риски перед релизом
- повысить качество продукта
