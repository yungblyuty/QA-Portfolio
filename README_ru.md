# Структура портфолио

## Case Studies

Полные исследования процесса тестирования игровых систем мобильных игр.

Каждый Case Study демонстрирует полный цикл QA работы:

- анализ функциональности
- определение целей тестирования
- анализ продуктовых рисков
- выбор стратегии тестирования
- проектирование тестового покрытия
- выполнение исследовательского тестирования
- проверка аналитики
- анализ найденных проблем
- оценка влияния на качество продукта

Каждый Case Study содержит связанную тестовую документацию:

- Test Plan
- Test Cases
- Bug Reports
- Exploratory Testing Report
- Analytics Validation (при необходимости)

Документы Test Plan, Test Cases и Bug Reports оформлены в виде отдельных Excel-файлов и связаны с соответствующим Case Study.

# Представленные Case Studies

## In-Game Store & Monetization Systems

Тестирование внутриигрового магазина и систем монетизации.

Фокус тестирования:

- стабильность процесса покупки
- корректная выдача наград
- логика специальных предложений
- пользовательский опыт
- риски монетизации
- влияние дефектов на продукт

Документация:

- Case Study
- Test Plan
- Test Cases
- Bug Reports
- Exploratory Testing

Документация:

# Исследовательское тестирование

Исследовательское тестирование является частью каждого Case Study и показывает:

- поиск нестандартных сценариев
- анализ рисков
- проверку граничных условий
- исследование поведения сложных систем
- поиск проблем за пределами подготовленных тест-кейсов

---

# Проверка аналитики

Раздел демонстрирует мой подход к проверке аналитических систем:

- проверка отправки событий
- проверка корректности данных
- поиск отсутствующих событий
- поиск дублирующихся событий
- проверка последовательности событий
- анализ влияния ошибок аналитики на продуктовые решения

---

# Улучшение QA-процессов

Отдельный раздел, посвящённый инициативам по оптимизации процессов:

- улучшение взаимодействия между QA и разработкой
- сокращение цикла обратной связи
- повышение эффективности проверки исправлений
- улучшение процесса подготовки релизов
