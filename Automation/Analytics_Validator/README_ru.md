# Валидатор игровой аналитики

## Обзор проекта

**Game Analytics Validator** — проект по автоматизации тестирования, направленный на проверку корректности аналитических событий в мобильных играх.

Проект демонстрирует автоматизацию проверок, которые обычно выполняются при валидации игровой аналитики:

- игровые события (Live Events)
- системы прогрессии
- системы наград
- отслеживание действий игроков

Основная цель проекта — обеспечить достоверность аналитических данных и предотвратить попадание некорректной информации в продуктовую аналитику.

---

# Область тестирования

Автоматические проверки охватывают:

- наличие обязательных аналитических событий;
- корректную последовательность событий;
- обнаружение дублирующихся событий;
- ошибки отслеживания, связанные с получением наград.

---

# Тестовые данные

Аналитические события хранятся в формате JSON.

Пример события:

```json
{
  "event_name": "reward_claimed",
  "user_id": "user_001",
  "reward_type": "coins",
  "reward_amount": 100
}
```

# Автоматизированные тесты

## TC-AUTO-001. Проверка обязательных событий

Проверяет наличие всех обязательных аналитических событий.

Проверяемые события:

- event_started
- milestone_completed
- reward_claimed
- event_completed

---

## TC-AUTO-002. Поиск дублирующихся событий

Проверяет отсутствие дублирования аналитических событий.

Позволяет предотвратить:

- повторную отправку событий получения награды;
- искажение аналитических метрик;
- повреждение аналитических воронок.

---

## TC-AUTO-003. Проверка последовательности событий

Проверяет правильный порядок отправки аналитических событий.

Ожидаемый сценарий:

Игрок начинает событие

↓

Проходит этап

↓

Получает награду

↓

Завершает событие

Некорректные последовательности обнаруживаются автоматически.

# Ошибки, обнаруживаемые автоматизацией

## Дублирование событий получения награды

### Риск при ручном тестировании

При определённых сценариях завершения события информация о получении награды могла отправляться несколько раз.

### Автоматическая проверка

- обнаруживает повторные события получения награды;
- предотвращает искажение аналитики наград;
- защищает корректность аналитических воронок.

---

## Неполное отслеживание прогрессии

### Риск при ручном тестировании

Аналитика прогресса могла пропускать промежуточные этапы прохождения.

### Автоматическая проверка

- контролирует наличие обязательных событий прогрессии;
- обнаруживает отсутствующие аналитические данные;
- повышает достоверность аналитики прогресса.

# Связь с опытом ручного тестирования

Этот проект основан на реальных задачах, возникающих при тестировании мобильных игр:

- проверка игровой аналитики;
- тестирование игровых событий (Live Events);
- тестирование систем прогрессии;
- проверка начисления наград.

Проект демонстрирует переход от ручной проверки аналитики к автоматизированному контролю качества данных.
