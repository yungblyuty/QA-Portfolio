# Проверка аналитики игровых событий и систем прогрессии

Проект: Мобильная casual-игра (Farm Simulator)

---

# Цель документа

Цели проверки аналитики:

- проверить корректность отслеживания аналитических событий
- убедиться в полноте аналитики прогрессии
- проверить корректность отслеживания наград
- предотвратить искажение продуктовых метрик
- минимизировать риски для продуктовой аналитики

---

# Основные области проверки аналитики

В рамках проверки анализировались следующие системы:

| Система | Проверка |
|---|---|
| Аналитика игровых событий | Отслеживание прогрессии |
| Аналитика наград | События получения наград |
| Аналитика монетизации | Сценарии покупок и наград |
| Аналитика прогрессии | Этапы прогрессии |
| Аналитика обучения | Отслеживание обучения игрока |

---

# Проверка аналитических событий

| Событие | Триггер | Проверка |
|---|---|---|
| Начало события | Запуск игрового события | Событие отправляется корректно |
| Завершение этапа | Завершение milestone-этапа | Корректный идентификатор этапа |
| Получение награды | Получение награды | Корректные данные награды |
| Полное завершение события | Полное завершение события | Корректное состояние завершения |
| Начало повторного прохождения | Запуск повторного прохождения | Корректный идентификатор уровня |
| Завершение повторного прохождения | Завершение повторного прохождения | Корректное количество звёзд |
| Начало обучения | Запуск обучения | Корректный идентификатор обучения |
| Завершение обучения | Завершение обучения | Корректное отслеживание завершения |

---

# Проверяемые параметры

Во время проверки аналитики анализировались:

- последовательность аналитических событий
- корректность данных событий
- дублирование аналитических событий
- отсутствие аналитических событий
- время отправки событий
- порядок прогрессии
- значения milestone-этапов
- значения наград
- идентификаторы уровней
- состояния завершения

---

# Основные сценарии проверки

---

## Проверка прогрессии

### Проверка:
- последовательной прогрессии
- одновременного завершения нескольких milestone-этапов
- обновлений прогрессии после повторного прохождения
- граничных состояний прогрессии

---

## Проверка наград

### Проверка:
- событий получения наград
- дублирования наград
- одновременного получения наград
- прерванных сценариев получения наград

---

## Проверка игровых событий

### Проверка:
- времени запуска события
- логики завершения события
- сценариев повторного запуска события
- синхронизации события

---

# Проверка нестандартных сценариев

Особое внимание уделялось:

- одновременным обновлениям прогрессии
- сценариям переподключения
- прерванным игровым сценариям
- принудительному закрытию приложения
- задержкам синхронизации
- многократному повторному запуску уровней

---

# Обнаруженные проблемы

---

## Дублирование аналитических событий получения наград

### Описание

При определённых сценариях завершения события аналитические события получения наград отправлялись повторно.

### Риски

- искажение метрик наград
- некорректная продуктовая аналитика
- повреждение аналитических воронок события

### Результат

Проблема была задокументирована и исправлена до релиза.

---

## Неполное отслеживание аналитики прогрессии

### Описание

При одновременном завершении нескольких milestone-этапов система аналитики пропускала промежуточные состояния прогрессии.

### Риски

- неполная аналитика прогрессии
- некорректные аналитические воронки
- недостоверные метрики прогрессии

### Результат

Проблема была эскалирована и исправлена до релиза.

---

# Результаты проверки

Проверка аналитики помогла:

- выявить нестабильные аналитические сценарии
- предотвратить повреждение аналитических воронок
- повысить надёжность продуктовых метрик
- улучшить качество отслеживания прогрессии
- минимизировать риски продуктовой аналитики

---

# Основные выводы

Проверка аналитики показала, что:

- системы прогрессии чувствительны к нестандартным сценариям
- одновременные обновления прогрессии требуют дополнительного тестового покрытия
- последовательность аналитических событий критична для корректной продуктовой аналитики
- исследовательская проверка аналитики помогает выявлять нестандартные проблемы отслеживания событий

# Итог

Проведённая проверка аналитики позволила:

- повысить надёжность аналитических систем
- улучшить качество продуктовых метрик
- предотвратить искажение аналитики прогрессии
- минимизировать риски для принятия продуктовых решений

Документирование проверки аналитики стало частью процесса оценки готовности релиза для систем игровых событий и механик прогрессии.
